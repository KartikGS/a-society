# A-Society: Project Log

## Scope Tags

| Tag | Meaning |
|---|---|
| `[S]` — Small — single-session maintenance change (one to two files) |
| `[M]` — Medium — multi-step change within a single phase |
| `[L]` — Large — multi-phase or multi-file strategic change |
| `[LIB]` — Library — creates or modifies `general/` content; scope additions require Owner approval |
| `[MAINT]` — Maintenance — a-docs change within Curator authority; no Owner proposal needed |
| `[ADR]` — Direction decision involved; requires Owner-level scope |
| `[RUNTIME]` — Runtime-root executable change; follows Executable Dev workflow when implementation is required |

Tags may be combined: `[M][LIB]`, `[S][MAINT]`, `[L][LIB][ADR]`.

## Current State

**Status:** Framework active at v37.0. `general/` is now an explicit two-tier library — universal layer at the `general/` root and category layer at `general/project-types/<type>/`. The a-docs scaffold manifest has moved out of `general/` and is now the runtime-owned `$A_SOCIETY_RUNTIME_ADOCS_MANIFEST` contract under `runtime/contracts/`; runtime initialization and runtime health checks use that same contract for required project `a-docs/` surfaces. Upstream feedback artifacts require generality classification (universal / category-shaped / project-specific) per framework-improvement candidate, and new `general/project-types/<type>/` category folders remain Owner-approved scope decisions. Standing executable model remains fully consolidated: `runtime/` is the sole executable root with the operator surface provided as a local web server and browser UI (project selector, Owner bootstrap chat, React Flow workflow graph). The runtime placement standard now reserves the root for the operator entry point plus package/config files, directs runtime-owned contracts to `runtime/contracts/`, and directs `runtime/src/` files into capability folders. The executable layer has four developer roles — Framework Services Developer, Orchestration Developer, UI Developer — plus the Technical Architect as design authority (under the executable project-type category in `general/`).
