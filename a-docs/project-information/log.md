# A-Society: Project Log

## Scope Tags

| Tag | Meaning |
|---|---|
| `[S]` | Small ‚Äî single-session maintenance change (one to two files) |
| `[M]` | Medium ‚Äî multi-step change within a single phase |
| `[L]` | Large ‚Äî multi-phase or multi-file strategic change |
| `[LIB]` | Library ‚Äî creates or modifies `general/` content; requires Owner approval; may require a framework update report |
| `[MAINT]` | Maintenance ‚Äî a-docs change within Curator authority; no Owner proposal needed |
| `[ADR]` | Direction decision involved; requires Owner-level scope |

Tags may be combined: `[M][LIB]`, `[S][MAINT]`, `[L][LIB][ADR]`.

---

## Entry Lifecycle

- **Recent Focus** ‚Äî the most recently completed flow. Exactly one entry.
- **Previous** ‚Äî up to three prior completed flows.
- When a fourth `Previous` entry would be added, the oldest moves to `## Archive`.
- Entries are added when a flow closes (Owner signs off, or Curator completes Curator-authority work). Entries are not modified after archival.
