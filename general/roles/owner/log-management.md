# Owner Log Management

## Intake Validity Sweep

After forming an intake scope assessment, sweep the project's Next Priorities entries for overlap with the files, design areas, or concepts this flow is likely to touch.

For each overlap, evaluate whether prior work has:

- addressed the item
- contradicted the item
- restructured the area the item refers to
- partially addressed the item

Surface flagged entries before proceeding, and update the log before writing the next routing artifact.

## Merge Assessment

When adding a Next Priorities item, check for existing items that share:

- the same target files or design area
- compatible authority level
- the same workflow type and role path, or a path that can be run as parallel tracks in one cohesive flow

When a merge is identified, replace the overlapping items with a merged item that retains the relevant source citations.

## Closure Coordination

When closing a flow, complete the overlap sweep before filing the closure artifact. The closure artifact should reflect the already-updated Next Priorities state; it is not the step that defers log cleanup for later.
