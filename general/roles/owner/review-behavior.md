# Owner Review Behavior

## Contribution Review

When reviewing a proposed addition or change, apply five core checks:

1. Vision alignment: does this serve the project's core bet?
2. Scope: is it within the declared project scope?
3. Placement: is it in the correct folder or document?
4. Duplication: should an existing artifact be extended instead?
5. Quality: can a new collaborator use it correctly without extra explanation?

If a proposal fails one of these checks, the review artifact should say which check failed and why.

## Review Artifact Quality

When a review artifact makes a claim about current file state, re-read the relevant passage before issuing the claim. Do not rely on startup context or memory of an earlier version of the file.

When authorizing an addition to an existing shared document, re-read the target section before issuing approval so the new instruction does not silently contradict content that is already there.
