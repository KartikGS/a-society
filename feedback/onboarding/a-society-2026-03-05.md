## Patterns Observed

**Self-referential initialization.** A-Society initializing its own `a-docs/` is structurally identical to any other initialization — same phases, same documents, same quality bar. The only difference is that A-Society's existing `a-docs/` was already partially built, which makes this a completion run rather than a full bootstrap. The Initializer protocol handled this without modification; the protocol doesn't assume a blank slate.

**Two-role project pattern.** Many real projects may have only two roles (e.g., Owner + Curator, or Director + Executor). The improvement protocol's three-role model (planning, requirements, improvement agent) is weighted toward software teams. A minimal role-mapping guide for smaller projects would reduce judgment calls during initialization.
